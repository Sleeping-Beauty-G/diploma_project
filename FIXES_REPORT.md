# 🛠 FIXES REPORT — Retail System API

## 📌 Общая информация

Данный документ описывает внесённые исправления в рамках доработки дипломного проекта **Retail System API** после проверки преподавателем.

Все замечания были проанализированы и устранены с целью полного соответствия требованиям ТЗ и рекомендациям ревью.

---

# ✅ 1. Корзина (Cart API) — приведена к REST стандарту

### ❗ Проблема:

* использовалась нестандартная логика добавления/удаления товаров
* отсутствовали PATCH / DELETE операции

### ✔ Исправлено:

* реализован полноценный REST ресурс `CartItem`
* добавлены методы:

```
GET    /api/cart/items/
POST   /api/cart/items/
PATCH  /api/cart/items/{id}/
DELETE /api/cart/items/{id}/
```

✔ теперь корзина полностью соответствует REST архитектуре

---

# ✅ 2. Адреса доставки (Address API)

### ❗ Проблема:

* отсутствовали API endpoints для управления адресами

### ✔ Исправлено:

Добавлен `AddressViewSet`:

```
GET    /api/addresses/
POST   /api/addresses/
PATCH  /api/addresses/{id}/
DELETE /api/addresses/{id}/
```

✔ адреса привязаны к текущему пользователю (`request.user`)
✔ добавлена защита `IsAuthenticated`

---

# ✅ 3. Заказы (Orders API)

### ❗ Проблема:

* отсутствовал `GET /api/orders/`
* неполная реализация OrderViewSet

### ✔ Исправлено:

* добавлен `list()` и `retrieve()` через ModelViewSet
* реализована корректная работа заказов пользователя

```
GET /api/orders/
GET /api/orders/{id}/
```

# ✅ 6. Роли пользователей и безопасность

### ❗ Проблема:

* роли (client / supplier / admin) не использовались корректно

### ✔ Исправлено:

* добавлены кастомные permission classes:

  * `IsSupplierOrAdmin`
  * `IsOwnerOrReadOnly`
* ограничен доступ к:

  * продуктам поставщика
  * admin-операциям
  * заказам других пользователей

✔ реализован Role-Based Access Control (RBAC)
