## 🚨 1. Исправление архитектуры API

### ❌ Было:

* Использование `ViewSet` с кастомными endpoint’ами (`/cart/add/`, `/remove/`)
* Нарушение REST-принципов
* Дублирование логики (`checkout_views.py` и `views.py`)

### ✅ Стало:

* Внедрён `ModelViewSet` для работы с ресурсами
* Корзина реализована как ресурс **CartItem**
* Удалено дублирование логики
* API приведён к REST-стандарту

📌 Пример:

```
GET    /api/cart/items/
POST   /api/cart/items/
PATCH  /api/cart/items/{id}/
DELETE /api/cart/items/{id}/
```

## 🛒 2. Полная переработка корзины

### ❌ Было:

* Нестандартные endpoint’ы (`add/remove/update`)
* Нарушение REST

### ✅ Стало:

## 📦 3. Исправление заказов (Order)

### ❌ Было:

* Отсутствовал `GET /api/orders/`
* Не было изменения статуса заказа
* Логика заказа частично дублировалась

### ✅ Стало:

* Добавлен endpoint:

```
GET /api/orders/
```

* Добавлен:

```
PATCH /api/orders/{id}/change_status/
```

* Вся логика сосредоточена в `OrderViewSet`

---

## 🧾 4. Улучшение checkout логики

### ❌ Проблемы:

* Возможны race condition
* Нет гарантии целостности данных
* Неполная проверка остатков

### ✅ Исправлено:

* Используется `transaction.atomic()`
* Добавлен `select_for_update()` (блокировка строк)
* Проверка остатков (`quantity__gte`)
* Очистка корзины после успешного заказа

📌 Гарантия:
заказ либо создаётся полностью, либо не создаётся вообще (ACID)
