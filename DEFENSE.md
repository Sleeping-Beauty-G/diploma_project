# 🚀 Retail System API — DEFENSE PACKAGE



---

# 📌 1. КРАТКОЕ ОПИСАНИЕ ПРОЕКТА

Retail System API — backend-сервис для автоматизации закупок в розничной сети.

Проект реализует REST API, позволяющий:

* управлять пользователями с ролями (client / supplier / admin)
* работать с каталогом товаров и предложениями поставщиков
* оформлять заказы через корзину
* обрабатывать заказы и менять их статусы
* импортировать товары из YAML
* выполнять фоновые задачи (Celery)
* отправлять email-уведомления

---

# 🧠 2. АРХИТЕКТУРА ПРОЕКТА (что говорить преподавателю)

Проект построен по слоистой архитектуре:

* Django REST Framework — API слой
* Service logic — бизнес-логика внутри viewsets
* PostgreSQL — хранение данных
* Redis — брокер сообщений
* Celery — асинхронные задачи
* Docker — контейнеризация

---

# ⚙️ 3. КЛЮЧЕВЫЕ ТЕХНИЧЕСКИЕ РЕШЕНИЯ

## ✔ Разделение ответственности

* models — структура данных
* serializers — валидация и DTO
* views — бизнес-логика API
* tasks — асинхронные операции

---

## ✔ Асинхронность (Celery)

Используется для:

* импорта YAML товаров
* отправки email уведомлений

---

## ✔ Signals

После регистрации пользователя автоматически:

* создаётся email уведомление
* отправляется welcome письмо

👉 Это реализация событийной архитектуры (event-driven approach)

---

## ✔ Безопасность

* JWT Authentication (SimpleJWT)
* Role-Based Access Control (RBAC)
* IsAuthenticated permissions
* защита supplier/admin endpoints

---

## ✔ Защита от гонок

* transaction.atomic
* select_for_update

👉 предотвращает конфликт списания остатков

---

# 📊 10. ОТВЕТИТЫ НА ВОПРОСЫ 

## ❓ Почему signals?

👉 “Для автоматической отправки email после регистрации, чтобы отделить бизнес-логику от views.”

---

## ❓ Почему Celery?

👉 “Для асинхронной обработки медленных операций: импорт и email.”

---

## ❓ Почему REST CartItem?

👉 “Это правильная REST модель — корзина как ресурс, а не action-based API.”

---

## ❓ Как обеспечивается безопасность?

👉 “JWT + RBAC + permission classes + transaction safety.”

---

## ❓ Что делает проект production-ready?

* Docker
* Celery
* PostgreSQL
* разделение слоёв
* API документация Swagger
